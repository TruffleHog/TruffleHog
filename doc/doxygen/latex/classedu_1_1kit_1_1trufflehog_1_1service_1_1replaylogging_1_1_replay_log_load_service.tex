\subsection{Detailed Description}
The \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log_load_service}{Replay\+Log\+Load\+Service} controls the playback of the playback graph (the old graph, generated from the 
\begin{DoxyCode}
ReplayLog 
\end{DoxyCode}
 s). It runs in its own thread, however most of its methods are called in other threads where the actual playback is controlled. Instead the \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log_load_service}{Replay\+Log\+Load\+Service} makes sure there is always data to playback in the background by loading and buffering \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log}{Replay\+Log} objects, and dispatching the commands of the \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log}{Replay\+Log} objects. (The \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log_load_service}{Replay\+Log\+Load\+Service} can do this since it is also a Notifier).

Starts the replay log load service. 

The \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log_load_service}{Replay\+Log\+Load\+Service} runs in its own thread to buffer \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log}{Replay\+Log} objects in the background so that when the graph playback function is activated, the old graph can immediately be started. It also notifies the playback executor with the old commands found in the \hyperlink{classedu_1_1kit_1_1trufflehog_1_1service_1_1replaylogging_1_1_replay_log}{Replay\+Log} objects so that the executor can update the playback graph.
